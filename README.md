# GPT Taxonomy Generator

CLI-инструмент для генерации иерархии категорий на базе модели DeepSeek.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

## Запуск

```bash
export DEEPSEEK_API_KEY=...
python -m src.cli --root "Ремонт и строительство" --depth 5 --breadth 8
```

## Запуск в Docker

Ниже описан полный цикл работы с Docker: от сборки образа до удаления контейнера после выполнения задачи.

### 1. Сборка образа

```bash
docker build -t category-generator .
```

Команда создаст образ `category-generator`, содержащий все зависимости приложения.

### 2. Подготовка файла с токеном API

1. Создайте файл `.env.docker` рядом с `Dockerfile`.
2. Добавьте в него ваш токен доступа к DeepSeek:
   ```bash
   echo "DEEPSEEK_API_KEY=ваш_секретный_токен" > .env.docker
   ```

Файл `.env.docker` не попадает в образ и добавлен в `.dockerignore`, поэтому его можно безопасно хранить локально. Если вы предпочитаете не создавать файл, можно передать токен напрямую через флаг `-e DEEPSEEK_API_KEY=...`, однако это оставит его в истории shell.

### 3. Запуск контейнера

```bash
docker run --rm \
  --name category-generator-run \
  --env-file ./.env.docker \
  -v "$(pwd)/out:/app/out" \
  category-generator \
  --root "Ремонт и строительство" \
  --depth 5 \
  --breadth 8
```

- Параметр `--rm` автоматически удалит контейнер после завершения работы приложения.
- Флаг `--name` задаёт имя контейнера и упростит управление им, если вы решите запускать его без `--rm`.
- Монтаж каталога `out` позволяет сохранять результаты на хосте. При необходимости измените путь до локальной папки.

Все аргументы после имени образа передаются CLI-приложению. Вы можете добавить другие параметры `src.cli`, например `--lang`, `--existing` или `--token-budget`.

### 4. Повторный запуск или ручное удаление контейнера

Если вы запускали контейнер без флага `--rm`, остановите и удалите его вручную:

```bash
docker stop category-generator-run
docker rm category-generator-run
```

После выполнения этих команд контейнер будет полностью удалён, а локальная папка `out/` сохранит сгенерированные файлы категорий.
